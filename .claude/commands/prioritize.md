# /prioritize 명령어

GAPEVI 우선순위(Prioritize) 단계를 실행합니다.

---

## 사용법

```
/prioritize [작업 목록]
```

---

## 목적

작업을 분해하고 실행 순서를 결정합니다.

---

## 실행 내용

### 1. 작업 분해

| 활동 | 도구 | 설명 |
|------|------|------|
| 세부 작업 생성 | TodoWrite | 구체적 태스크 목록 |
| 의존성 파악 | - | 작업 간 순서 관계 |

### 2. 실행 순서 결정

**게임 개발 순서** (기본):
```
1. 타입 정의 (types/)
2. 엔티티 구현 (entities/)
3. 시스템 구현 (systems/)
4. 씬 통합 (scenes/)
5. UI 연결 (ui/)
6. 테스트 코드 작성 (*.test.ts)
```

### 3. 우선순위 부여

| 우선순위 | 기준 |
|---------|------|
| 높음 | 다른 작업의 전제 조건 |
| 중간 | 핵심 기능 |
| 낮음 | 개선/선택 사항 |

---

## 출력 형식

```markdown
## 작업 계획

### 작업 목록

| # | 작업 | 우선순위 | 에이전트 | 상태 |
|---|------|---------|----------|------|
| 1 | [작업명] | 높음 | **Entity** | [ ] |
| 2 | [작업명] | 중간 | **System** | [ ] |
| 3 | [작업명] | 낮음 | **Tester** | [ ] |

### 실행 순서
```
[1] → [2] → [3]
     ↘     ↗
       [4]
```

### 예상 결과
- [완료 후 기대 상태]
```

---

## 작업 상태

| 상태 | 기호 | 설명 |
|------|------|------|
| 대기 | `[ ]` | 아직 시작 안 함 |
| 진행중 | `[~]` | 현재 작업 중 |
| 완료 | `[x]` | 완료됨 |
| 차단 | `[!]` | 다른 작업 대기 |

---

## 다음 단계

`/prioritize` 완료 후 `/execute` 실행

---

## 조합 예시

```bash
# 단독 실행
/prioritize 레벨업 시스템 구현

# 전체 사이클에서 사용
/new-feature  # 내부적으로 ... → /prioritize → ... 호출
```
