# /verify 명령어

GAPEVI 검증(Validate) 단계를 실행합니다.

---

## 사용법

```
/verify [검증 대상]
```

---

## 목적

구현 결과를 검증하고 품질을 확인합니다.

---

## 실행 내용

### 1. 자동화 테스트

| 항목 | 명령어 | 통과 기준 |
|------|--------|----------|
| 단위 테스트 | `npm run test` | 모든 테스트 통과 |
| 타입 체크 | `npx tsc --noEmit` | 오류 0개 |
| 빌드 | `npm run build` | 성공 |

### 2. Quality Gate

```bash
# 자동화 전체 실행
make qa-auto
```

| 게이트 | 기준 |
|--------|------|
| 테스트 | 100% 통과 |
| 타입 | 오류 0개 |
| 빌드 | 성공 |
| 커버리지 | 80% 이상 (systems/) |

### 3. 수동 테스트 (필요 시)

| 상황 | 시나리오 |
|------|----------|
| 커밋 전 | `smoke.md` |
| 전투 변경 | `battle.md` |
| 이동/맵 변경 | `exploration.md` |
| UI 변경 | `ui.md` |
| PR 전 | `regression.md` |

---

## 출력 형식

```markdown
## 검증 결과

### 자동화 테스트
| 항목 | 결과 | 상세 |
|------|------|------|
| 단위 테스트 | ✅/❌ | N/N 통과 |
| 타입 체크 | ✅/❌ | N errors |
| 빌드 | ✅/❌ | - |

### Quality Gate
- [x] 테스트 통과
- [x] 타입 오류 없음
- [x] 빌드 성공

### 수동 테스트 (필요 시)
- 권장 시나리오: [시나리오명]
```

---

## 조합 예시

```bash
# 단독 실행
/verify

# 빠른 검증 (기존 명령어)
/validate  # 타입 체크 + 빌드만

# QA 전체 실행
/qa  # 자동화 + 수동 테스트 안내

# 전체 사이클에서 사용
/new-feature  # 내부적으로 ... → /verify → ... 호출
```

---

## 관련 명령어

| 명령어 | 설명 |
|--------|------|
| `/validate` | 빠른 검증 (타입+빌드) |
| `/qa-auto` | 자동화 테스트 |
| `/qa` | 전체 QA 사이클 |
